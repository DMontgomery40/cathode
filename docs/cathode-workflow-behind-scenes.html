Lights-out video manufacturing

The dark factory: agents prompting agents prompting agents.

Cathode looks like a one-button video maker. Behind the glass it's a delegation chain: one agent calls Cathode's MCP, which spawns a director agent, which writes scenes for image and voice models, whose output is judged by another agent, which prompts more agents to repair what failed — until an MP4 falls out the end. Almost no human in the loop. The brief enters as English; the artifact leaves as .mp4. Everything in between is one model talking to the next.

01

Who staffs each station

4 operators

Every box on Cathode's floor is run by either a Claude agent, a model call, or deterministic code. The same pipeline can be entered by a human or, more interestingly, by another agent — at which point the chain has no human at any point, just delegation all the way down.

caller agent

External agent calls in

Another Claude (or any MCP client) invokes make_video with intent, sources, footage, and depth. From here, the brief is now flowing through agents who haven't met the human.

director agent

Claude writes the scenes

The director agent receives the normalized brief, decides composition, drafts narration, and emits a scene list with visual prompts. It is itself a Claude call wrapped in repo prompt scaffolding.

scene models

Image & voice models execute

For every scene the director invented, prompts go out: image gen (local, Replicate, Codex) and TTS (Kokoro, ElevenLabs, OpenAI). They don't reason — they render whatever Claude told them to.

reviewer agent

Another Claude grades the output

A second-pass agent inspects rendered scenes, finds bad text, mismatched visuals, or broken framing — then writes new prompts and sends the work back through the model stations. The factory loops until accepted.

06

Review & repair

agent

A second Claude is called per-scene to grade what the models produced. It can demand exact-text repairs or rewrite the visual prompt and send the scene back to the model. The plan mutates; the rerender fires.

core.scene_review · _attempt_text_repairs()
07

Render

code

Classic image/video projects use ffmpeg assembly. Motion, hybrid, native, overlay, or transition scenes use a Remotion manifest. The decision is resolved by code, not chosen by an agent.

Yet another Claude pass watches the assembled video slide by slide. It can still order repairs, which propagate back to the asset stations and rerender. This is the deepest the call stack gets.

Everything orbits plan.json

The plan is the persisted record of what every agent decided. Reviewers mutate it; reruns reload it; the next agent down the chain trusts it. Loading or saving also backfills schema and refreshes cost metadata.

Provider UI is env-driven

Cathode only surfaces models that are actually configured. Kokoro is always available for local voice. Anthropic is required because Claude is the agent at every reasoning station.

Render backend is resolved, not guessed

Forced strategies win first. Otherwise motion scenes, native treatments, overlays, transitions, or hybrid/motion modes select Remotion. Plain image/video projects stay on ffmpeg. No agent is involved in this step.

force_ffmpeg                    -> ffmpeg
force_remotion                  -> remotion
motion scene                    -> remotion
native / overlay / transition   -> remotion
composition_mode = hybrid       -> remotion
otherwise                       -> ffmpeg

Failures are meant to be recoverable

Jobs return partial_success with retryable hints so a calling agent knows what to repair. Missing assets block final render honestly. Over-budget runs stop after storyboard and require an explicit rerun the caller has to authorize.

status:        queued | running | succeeded |
               partial_success | failed | cancelled

current_stage: storyboard | assets | render

suggestion:    "Review failed scenes or provider
                config, then rerun assets."

07

Mental model

6 contracts

The useful way to debug Cathode is to ask which contract between adjacent agents has — or hasn't — been satisfied yet.

Brief contract

Did the caller agent give the director enough audience, source material, and constraints to write scenes without guessing?

Plan contract

Does plan.json have normalized scenes with manifestation, composition, narration, and asset-path fields the next station can consume?

Provider contract

Are the LLM, image, video, voice, and render models the director picked actually reachable in this local environment?

Asset contract

For every scene, did the model station return a usable visual, and does narration exist unless the scene is intentionally clip-audio?

Render contract

Does the chosen backend match the scene requirements, and can it reach all project media through paths or API URLs?

Review contract

Did the reviewer agent leave any unresolved repairs, failed scene-review metadata, or partial-success retry path the caller must handle?
